# 2.1 Классический метод Гаусса (без выбора ведущего элемента)

## Метод Гаусса

Процесс решения системы состоит из двух этапов: прямого и обратного хода.

Во время прямого хода матрица приводится к верхнетреугольному виду. Для каждой строки вычисляется коэффициент:

m(i,k) = A[i][k] / A[k][k]

После чего элементы строки пересчитываются:

A[i][j] = A[i][j] - m(i,k) \* A[k][j]

b[i] = b[i] - m(i,k) \* b[k]

После завершения прямого хода выполняется обратный ход, при котором неизвестные вычисляются снизу вверх.

Недостатком метода является чувствительность к малым ведущим элементам, что может приводить к накоплению ошибок округления.

---

# 2.2 Метод Гаусса с выбором главного элемента

В данной модификации перед каждым шагом выбирается максимальный по модулю элемент текущего столбца. После этого строки матрицы меняются местами.

Такой подход позволяет повысить численную устойчивость метода и уменьшить влияние ошибок округления.

Дальнейшие шаги алгоритма совпадают с классическим методом Гаусса.

# 2.3 LU-разложение

Матрица A представляется в виде произведения:

A = LU

где:

- L — нижнетреугольная матрица;
- U — верхнетреугольная матрица.

После разложения решение системы сводится к двум этапам:

1. Решение Ly = b (прямая подстановка)
2. Решение Ux = y (обратная подстановка)

Главное преимущество метода заключается в том, что разложение матрицы выполняется только один раз. При изменении правой части требуется выполнять только подстановки, что существенно ускоряет вычисления.

---

## LU-разложение

Матрица представляется в виде:

A = LU

где:

- L — нижнетреугольная матрица;
- U — верхнетреугольная матрица.

После разложения решение системы сводится к двум подстановкам:

1. Ly = b
2. Ux = y

Главное преимущество LU-разложения — возможность многократного решения систем с одной матрицей и разными правыми частями.

# 3. Метрики оценки

## 3.1 Евклидова норма

Для вектора v евклидова норма вычисляется по формуле:

||v|| = sqrt(sum(v[i]^2))

## 3.2 Относительная погрешность

Относительная погрешность вычисляется как:

||x\_real - x|| / ||x\_real||

## 3.3 Невязка

# Теоретическая сложность алгоритмов

| Метод                             | Теоретическая сложность |
| --------------------------------- | ----------------------- |
| Метод Гаусса                      | O(n^3)                  |
| Метод Гаусса с pivot              | O(n^3)                  |
| LU-разложение                     | O(n^3)                  |
| Решение через LU после разложения | O(n^2)                  |

Из таблицы видно, что методы Гаусса и LU-разложение имеют одинаковую асимптотическую сложность при решении одной системы. Однако после выполнения LU-разложения дальнейшее решение систем с новыми правыми частями выполняется значительно быстрее, поскольку требует только прямой и обратной подстановок.

# Анализ полученных результатов

Проведённые эксперименты подтвердили ожидаемые теоретические свойства реализованных методов.

## Сравнение времени работы

Для одной системы все методы показывают близкое время выполнения. Это объясняется тем, что основная вычислительная нагрузка связана с приведением матрицы к треугольному виду, имеющим кубическую сложность.

Метод Гаусса с выбором ведущего элемента работает немного медленнее классического метода из-за необходимости поиска максимального элемента в столбце.

LU-разложение требует дополнительных вычислений на этапе факторизации матрицы, однако время последующих подстановок оказывается крайне малым.

---

## Множественные правые части

При увеличении числа правых частей преимущество LU-разложения становится особенно заметным.

Для метода Гаусса каждая новая система требует повторного выполнения полного прямого и обратного хода.

В случае LU-разложения факторизация матрицы выполняется один раз, после чего решение каждой новой системы требует только решения двух треугольных систем.

Эксперименты показали, что при большом количестве правых частей суммарное время LU-разложения оказывается значительно меньше времени повторного решения методом Гаусса.

---

## Матрица Гильберта

Матрица Гильберта является плохо обусловленной, поэтому даже небольшие ошибки округления начинают быстро накапливаться при увеличении размерности.

Полученные результаты показывают:

- рост относительной ошибки при увеличении размерности матрицы;
- улучшение устойчивости при использовании pivoting;
- сохранение малой невязки даже при больших ошибках решения.

Последний факт особенно важен, поскольку демонстрирует различие между невязкой и реальной точностью найденного решения.
